### 首先理想定义模型
输入：自然语言有限长度序列（约100k） 
输出：自然语言有限长度序列 
约束：类似图灵定义。即便是人类来做输出，也不可能比模型本身做得更好。也就是模型的输出没有任何不符合人类所能理解的逻辑的事情（例如不能正常计数等）。

由于注意力机制本身的设计，模型的上下文是不能无限扩张的。

### Agent工程
通过定义和切换上下文，使得模型发挥问答以外的能力。Agent工程是有相当多空间的，虽然大模型在各项考试中全面碾压人类，但它在复杂工程中完全无法替代人类。例如在rustc或者Linux源码中，agent并没有完全接管。如果agent真的像考试中那样全面碾压人类，我们应该有Austc或者Ainux这样纯agent设计的系统才对。

人类由于其生物大脑独有的优势，有很强的上下文压缩、联想和洞察能力，能够统筹全局建立认知结构、通过认知结构压缩信息、根据现实的反馈触发认知结构改变。即使人脑本身的计算能力极差，记忆力也极差，非常容易出现幻觉，但人类依然可以完成复杂的工程。人类可以隐含地学会某些东西并且触类旁通，例如理解Rust设计的人，会自然地建立认知结构，并感觉Linux内核设计中不合理的地方，而且如果他真的根据这个认知进行改进，大概率是有效的改进。而这个认知并不是完全可以由自然语言描述的。或者说，描述认知结构是我们现有自然语言所不具备的能力，未来LLM还得发明新的、人类无法理解的词汇和语法，才能准确地记录认知结构。

由于我们前文中提到的上下文限制问题，大模型本身无法感知到上下文以外的信息，自身会在行动中因为认知结构混乱而出现错误操作。认知结构是一种信息压缩和降维的方法，没有这种压缩，行动必然是混乱的。而根据认知结构来压缩必然依赖于偏见，或者我们可以认为认知结构就是一种被现实部分证明有效的偏见，比如Rust设计原则中正确性>性能>多支持>高产>透明>通用。这是一种恰好被证明有效的偏见。

在当前这个模型能力本身很弱、容易乱跑的时代，我们可以说agent工程就是harness工程，我们必须要防止模型乱跑。但是从更长远来说，agent工程是信息压缩的工程，是建立认知结构、行动观察、触发认知改变的工程。我们需要解决的问题是，如何在有限的上下文空间中，完成只有人类才能完成的巨大的复杂问题。Agent工程是在有限上下文窗口和无状态基础模型之上，构造可扩展认知系统（Scalable Cognitive Systems）的学科。

Agent工程是重要的，agent设计所主导的信息压缩模式，和模型本身必须是配套的。认知结构只是一种压缩模式，或许还会有别的模式。因为模型通过agent与世界交互，生成训练语料来进化。没有agent，模型本身没有足够的边缘训练语料。这些语料是具体的、详细的，避免模型说假大空、拥有正确的话。

Agent和Prompt不可能教会LLM做任何事情，除非这件事被写入训练数据中，我们通过组织方法来记录下LLM验证了能做的事情，及时召回上下文来达到效果。
